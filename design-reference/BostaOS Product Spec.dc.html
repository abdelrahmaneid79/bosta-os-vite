Every screen is URL-addressable so WhatsApp links open the exact view. Filters persist in the query string.

04

Interaction system

Every interactive element obeys the same state machine, gestures and feedback. Defined once; referenced by every component above.

4.1 · ELEMENT STATES (all interactives)
STATETREATMENT
restbase token colours
hover (d)+6% lightness · 120ms
pressscale .97 · 90ms
focus-visible2px --brand ring, 2px offset
active/selected--brand fill or 2px --brand underline
loadingspinner replaces label, width locked
disabled40% opacity, no pointer
4.2 · GESTURES & TRANSITIONS
TRIGGERRESULT
tap ＋QuickAddSheet slides up 240ms
tap StatCardcross-fade nav to drill target
tap LedgerRowDetailDrawer in from right/bottom
swipe-left row (m)reveal Edit · Delete
pull-down (m)refresh → sync
save recordsheet dismiss + number count-up + toast(undo)
deleteConfirmModal → optimistic remove + toast(undo 4s)
FEEDBACK LATENCY
<100ms optimistic UI on every write. Spinner only after 400ms. Skeleton only after 200ms.
VALIDATION
Inline, on blur. Error text under field in --down. Save disabled until amount > 0. Never a blocking alert.
UNDO-FIRST
Destructive actions are reversible for 4s via toast rather than guarded by friction. One ConfirmModal only for permanent deletes.
